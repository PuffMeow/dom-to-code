## 为什么

大多数中大型项目并不是一两个前端去开发的，可能有些不是你编写的业务你都不知道源码在哪里,并且随着项目迭代越来越多，组件不断增加。

如果给你个新需求或者再原来基础上改 bug，想快速定位到某个文件或某条代码，要么问同事 要么全局搜，其实这并不是一件特别高效的方法。

有了 dom-to-code， 可以让你通过浏览器 dom 直接到源代码，是不是方便很多
